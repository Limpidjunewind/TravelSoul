# Nomie 前端设计文档

> 基于 [PRD](./prd-travel-agent.md) 的前端布局与交互设计。
> 技术栈：React + JavaScript (JSX) + CSS pixel art theme（无 Bootstrap）

---

## 整体结构

应用采用**侧栏 + 主内容区**的经典布局，类似 ChatGPT / Claude 的界面结构。

```
┌──────────────┬──────────────────────────────┐
│              │                              │
│    侧栏      │         主内容区              │
│   (固定)     │    （根据页面/状态变化）        │
│              │                              │
└──────────────┴──────────────────────────────┘
```

侧栏始终存在（可折叠），主内容区根据当前页面和 Agent 状态动态切换。

---

## 侧栏设计

侧栏包含三个区域：新建按钮、收藏分组、历史对话列表。

```
┌──────────────┐
│ 🆕 新建旅行   │  ← 主操作按钮
│ ─────────── │
│ ⭐ 收藏方案   │  ← 可折叠分组
│   ✈️ 春秋东京  │    点击 → 主内容区显示纯方案卡片
│   🏨 新宿酒店  │    点击 → 主内容区显示纯方案卡片
│ ─────────── │
│ 💬 历史对话   │  ← 可折叠分组
│   📅 5/1 东京  │    点击 → 加载该 session 完整记录
│   📅 4/20 曼谷 │    （聊天记录 + 方案结果）
│   📅 4/15 首尔 │
│              │
│              │
│ ─────────── │
│ ⚙️ 设置 / 语言 │  ← 底部：设置、语言切换
└──────────────┘
```

**交互说明：**
- 「新建旅行」点击后，主内容区进入全屏聊天模式
- 收藏分组：展示用户收藏的具体方案（某个机票、某个酒店），点击后主内容区只显示方案卡片，无聊天记录
- 历史对话：每条是一个 session，点击后加载完整的聊天记录 + 方案结果
- 侧栏可折叠收起（移动端默认收起）

## 页面二：对话页（核心页）

对话页是整个应用的核心，有**三个阶段**，布局动态切换。

### 阶段一：聊天阶段（Agent 未执行）

用户正在跟 Agent 对话描述需求，此时体验跟普通 chatbot 一样。

```
┌──────────┬──────────────────────────────────┐
│          │                                  │
│  侧栏    │          全屏聊天窗口              │
│          │                                  │
│          │  🤖: 你好！请告诉我你的旅行计划～    │
│          │                                  │
│          │  👤: 我想五一去日本东京玩5天         │
│          │                                  │
│          │  🤖: 出发城市是哪里？               │
│          │                                  │
│          │  👤: 上海                          │
│          │                                  │
│          │                                  │
│          │  ┌──────────────────────────┐     │
│          │  │ 输入你的旅行需求...    [发送] │     │
│          │  └──────────────────────────┘     │
└──────────┴──────────────────────────────────┘
```

- 主内容区 100% 给聊天窗口
- 标准 chatbot 体验，消息气泡上下排列
- Agent 对话确认用户需求（目的地、时间、人数、预算、偏好）

### 阶段二：Agent 执行阶段

Agent 确认需求后开始执行 Deep Research，布局动态切换为**左窄右宽**分栏。

```
┌──────────┬─────────┬────────────────────────────────┐
│          │ 聊天    │      Agent 工作可视化面板         │
│  侧栏    │(20-30%) │         (60-80%)                │
│          │         │                                 │
│          │ 🤖:好的  │                                 │
│          │ 开始搜索 │    ✈️              🏨            │
│          │ 啦！    │  [搜索中]        [搜索中]        │
│          │         │  携程...       Booking...       │
│          │         │                                 │
│          │         │         📋                      │
│          │         │       [等待中]                   │
│          │         │                                 │
│          │         │              🔍                 │
│          │         │           [搜索中]               │
│          │         │          Google...               │
│          │         │                                 │
│          │         │  （空间散布式，参考 Pixel Agents）  │
│          │         │                                 │
│          │ [停止🛑] │                                 │
└──────────┴─────────┴────────────────────────────────┘
```

**聊天窗口（左侧 20-30%）：**
- 压缩显示，保留关键对话
- 底部输入框变为「停止」按钮（执行中不能发新消息，V1 不支持中途干预）
- Agent 可能会在聊天里发进度摘要（如"已找到 3 个航班"）

**Agent 可视化面板（右侧 60-80%）：**
- 采用 **2x2 grid 规整布局**，像素办公室背景图（office_bg.webp）
- 每个 sub-agent 是一张像素角色卡片（pixel sprite + 名称 + 状态 + 步骤列表）
- 角色状态：idle（灰色）→ working（蓝色/动画）→ done（绿色）→ error（红色）
- 搜索中状态有轻微跳动 CSS 动画
- 像素风素材来源：Star-Office-UI 项目（guest_role sprite sheets，裁剪首帧 + 3x 放大）

**点击展开详细日志：**

```
┌─────────────────────────────────┐
│ ✈️ 机票搜索 Agent    [搜索中...]  │
│ ────────────────────────────── │
│  ✅ 打开 Google Flights         │
│  ✅ 搜索 上海→东京 4/30-5/4      │
│  ✅ 找到 12 个航班结果           │
│  🔄 正在打开携程...              │  ← 当前步骤，转圈动画
│  ⬜ 搜索携程航班                 │  ← 未执行，灰色
│  ⬜ 比较价格，筛选最优方案        │
└─────────────────────────────────┘
```

- 点击 Agent 卡片展开/收起详细步骤
- 步骤状态：✅ 已完成 → 🔄 进行中 → ⬜ 等待中
- 步骤描述为自然语言，不是代码

**方案结果面板包含四个区块（可滚动）：**
1. **机票推荐** — 多个方案卡片，显示航班号、价格、时间，附「查看链接」和「收藏」按钮
2. **酒店推荐** — 多个方案卡片，显示名称、价格、位置，附链接和收藏
3. **行程表** — 按天展示行程安排
4. **注意事项** — 签证、天气、交通等提醒

**交互说明：**
- 「查看链接」跳转到真实预订平台（或触发 Agent 代操作）
- 「⭐收藏」将单个方案（某个机票/某个酒店）加入侧栏收藏分组
- 聊天窗口恢复输入框（用户可以开始新的对话或评论结果）

## 视觉风格

### 整体基调
**像素风（Pixel Art）**，参考 Star-Office-UI 项目。温暖复古的像素办公室风格，轻松有趣但不幼稚。

### 配色（CSS 变量管理）
- `--color-primary: #A0522D` — warm brick brown（主色）
- `--color-primary-dark: #5C3D2E` — dark wood（边框/阴影）
- `--color-secondary: #5B8C5A` — plant green（完成状态/辅助）
- `--color-accent: #D4A843` — gold/lamp warm（强调/收藏）
- `--color-bg: #F5E6D0` — parchment cream（背景）
- Agent 状态色：working（蓝色）、done（绿色）、error（红色）、idle（灰色）

### 字体
- **Ark Pixel**（ark-pixel-12px-proportional）像素字体，Latin + 中文两套 woff2
- fallback: monospace

### 像素风格元素
- `--pixel-border: 3px solid` 硬边框，无 border-radius
- `--pixel-shadow: 4px 4px 0px` 右下方块阴影
- `image-rendering: pixelated` 全局像素渲染
- 按钮 hover 时 translate(2px, 2px) + 阴影消失的"按下"效果

### Agent 角色
- 每个 sub-agent 使用像素 sprite 角色图（从 Star-Office-UI guest_role sprite sheet 裁剪首帧，3x nearest-neighbor 放大至 96x96）
- 保留 emoji fallback（✈️ 🏨 📋 🔍）用于无 sprite 时显示
- 搜索中状态：轻微弹跳 CSS 动画
- 动效全部 CSS-only，无 JS 动画库

## 阶段切换动效

对话页三个阶段之间的切换需要平滑过渡：

```
聊天阶段 ──→ 执行阶段：
  聊天窗口从 100% 平滑压缩到 20-30%
  Agent 面板从右侧滑入

执行阶段 ──→ 结果阶段：
  Agent 工作卡片淡出
  方案结果卡片淡入（保持同一区域）

结果阶段 ──→ 新对话：
  方案面板滑出
  聊天窗口恢复 100%
```

使用 CSS transition 实现宽度和透明度的平滑变化。

## 技术实现要点

- **状态机驱动**：MainPage 使用显式状态转换表（PHASE_TRANSITIONS）管理 chat → executing → result 的 phase 切换，所有转换通过 `transitionPhase('EVENT')` 触发
- **Chat 为受控组件**：消息数据和发送逻辑由 MainPage 管理，Chat 只负责展示和输入；通过 `onSendMessage` 回调将用户输入传回父组件
- **Agent 执行时间线**：执行阶段通过 `buildExecutionStages()` 生成分阶段快照，按时间推进动画（目前为 mock，后续接真实 SSE 流）
- **React 组件结构**：侧栏、聊天窗口、Agent 面板、方案面板为独立组件
- **CSS pixel art theme**：全局 CSS 变量管理像素风主题
- **CSS 动效**：Agent 状态动画、面板切换过渡，全部 CSS-only
- **响应式**：移动端侧栏默认收起，对话页执行阶段改为上下布局（Agent 面板在上，聊天在下）
